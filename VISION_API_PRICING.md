# 💰 Google Vision API 計費說明

## 📊 計費方式

### 基本原則

1. **按圖片計費**：每處理一張圖片算一次
2. **按功能計費**：每使用一種功能（Feature）算一個單位
3. **分層計價**：使用量越大，單價越低

## 🧮 費用計算範例

### 範例 1：低使用量（你的情況）

**使用情境**：
- 每天處理 10 張匯款截圖
- 每月約 300 張圖片

**費用計算**：
```
300 張 < 1,000 張（免費額度）
總費用 = $0 USD ✅
```

**結論**：完全免費！

---

### 範例 2：中等使用量

**使用情境**：
- 每天處理 50 張圖片
- 每月約 1,500 張圖片

**費用計算**：
```
前 1,000 張 = $0（免費）
剩餘 500 張 = (500 / 1,000) × $1.50 = $0.75 USD
總費用 = $0.75 USD（約 NT$23）
```

---

### 範例 3：高使用量

**使用情境**：
- 每天處理 200 張圖片
- 每月約 6,000 張圖片

**費用計算**：
```
前 1,000 張 = $0（免費）
第 1,001-5,000 張 = 4 × $1.50 = $6.00
第 5,001-6,000 張 = (1,000 / 1,000) × $1.50 = $1.50
總費用 = $7.50 USD（約 NT$235）
```

## ⚠️ 重要提醒

### 1. 需要啟用計費

- ❗ 即使在免費額度內，也需要綁定信用卡
- ✅ 免費額度內不會扣款
- ⚠️ 超過免費額度才會收費

### 2. 計費單位

- 📸 每張圖片 = 1 次請求
- 🔢 PDF 檔案：每頁算 1 次請求
- 🎯 每個功能分別計費（同一張圖用多個功能會累加）

### 3. 計費精度

- 📊 以 1,000 次為計費區塊
- 🔢 最後一個區塊按比例計費
- 💡 例如：1,100 次 = 1 個完整區塊 + 0.1 個區塊
